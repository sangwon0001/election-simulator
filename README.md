# 전남도지사 선거 '쌍둥이 표' 몬테카를로 시뮬레이션

서로 다른 **두 투표소**에서 A후보 득표수와 B후보 득표수가 **동시에 똑같은**
'쌍둥이 투표소'가 전남 전체 투표소 중 한 쌍이라도 발생할 확률을 추정합니다.
즉 투표소 i, j 에 대해 `A_i == A_j` **그리고** `B_i == B_j` 인 충돌 — **생일 문제(birthday problem)** 구조입니다.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `twin_vote_sim.py` | Hybrid Monte Carlo 시뮬레이션 엔진 (벡터화) |
| `data_loader.py` | 선관위 엑셀 → 단위별 선거인수(N_i) 추출/전처리 (대안 경로) |
| `build_dataset.py` | 크롤링 JSON 22개 병합 → `jeonnam_units.csv`, `jeonnam_voters.npy` + 경험적 쌍둥이 확인 |
| `run_real.py` | 실제 전남 데이터 주입 시뮬 vs 경험적 관측 비교 |
| `jeonnam_46??.json` | playwright로 크롤링한 시군구별 개표단위 원본 (info.nec.go.kr) |

## 실제 데이터 검증 (제9회 지방선거, 전남도지사 / A=민형배, B=이정현)

중앙선관위 선거통계시스템에서 22개 시군구 개표단위(읍면동×구분)를 playwright로 크롤링.
실제 지지율: **민형배(A) 80.3% vs 이정현(B) 11.6%** (압승), 총 선거인 1,558,206명.

**경험적 관측 vs 시뮬레이션(실제 N_i·지지율 주입, 10만회):**

| 단위 정의 | 단위 수 | 실제 관측 쌍둥이 | 시뮬 기대 쌍수 | 시뮬 P(≥1쌍) |
|---|---|---|---|---|
| 동 단위(계) | 297 | **0쌍** | 0.21쌍 | 19.1% |
| 동×구분(선거일/관내사전) | 594 | **4쌍** | 3.15쌍 | 95.8% |

실제 관측된 4쌍(모두 관내사전투표): 여수 삼일동=신안 하의면(506,42), 장성 북하면=함평 엄다면(606,57),
보성 노동면=신안 팔금면(356,42), 화순 이양면=강진 병영면(444,46).

**모델 타당성**: 실제 N_i와 득표율만 넣었는데 시뮬 기대값(3.15쌍)이 실제 관측(4쌍)과 포아송 오차 내 일치 →
하이브리드 몬테카를로 모델이 현실을 잘 재현함이 검증됨. 큰 단위는 충돌 희박(동 0쌍), 소규모 단위가 많을수록 생일효과로 충돌 증가.

## 방법론

- **Divide & Conquer**: 전남을 투표소 단위로 분할, 난수 생성 후 전체 스캔하여 병합
- **Hybrid**: 상수(투표소 수 M, 유권자 수 N_i)는 실제값 / 변수(투표율·지지율·득표)는 확률분포
- **벡터화 최적화**: `(iter, station)` 2차원 배열을 청크로 처리 → 100k×860 ≈ 8,600만 시뮬레이션을 약 15초

### 다항분포 → 이항분포 분해 (득표 생성 최적화)

`np.random.multinomial`은 투표소마다 다른 `n`과 `p`를 동시에 벡터화하지 못합니다.
A, B 두 후보만 관심 대상이므로 다항분포를 이항분포로 분해합니다 (수학적으로 동치):

```
X_A ~ Binomial(V, p_A)
X_B ~ Binomial(V − X_A, p_B / (1 − p_A))
```

`rng.binomial`은 `n`, `p` 모두 배열을 받으므로 완전 벡터화됩니다.

### 충돌(쌍둥이 투표소) 탐지 — 생일 문제

각 반복마다 860개 투표소의 `(A표, B표)` 쌍 중 동일한 쌍이 존재하는지 봅니다.
2D 쌍을 단일 정수키로 인코딩 후 행별 정렬하여 인접 중복을 탐지 — O(M log M):

```
key = A · BIG + B          (BIG = max 선거인수 + 1 → B < BIG 보장, 충돌 없는 인코딩)
key.sort();  any(diff(key) == 0)   # 인접 동일 = (A,B) 쌍 충돌
```

`A`만 같거나 `B`만 같은 경우는 카운트되지 않음 (단위 테스트로 검증).

## 하이퍼파라미터 튜닝 (극단치 방지 수학)

### 1. 투표율 — Beta 분포

`turnout ~ Beta(a, b)`, 평균 μ, 집중도 κ = a + b 로 모수화:

```
a = μ · κ,   b = (1 − μ) · κ
분산 Var = μ(1−μ) / (κ + 1)
```

- **경계 보장**: Beta는 [0,1]에 갇혀 있어 **투표율 100% 초과가 구조적으로 불가능** (논리 오류 원천 차단)
- **κ 선택**: 투표소 간 투표율 표준편차를 σ_target 으로 잡고 역산
  `κ ≈ μ(1−μ)/σ_target² − 1`
  예) μ=0.60, 실제 투표소 표준편차 ≈ 3.5%p → κ ≈ 0.24/0.001225 − 1 ≈ **195** (코드 기본 200과 일치)
- κ를 너무 작게(<30) 잡으면 0%·100% 근처로 U자형 퍼짐 → 비현실적. κ ≥ 100 권장.

### 2. 지지율 — Dirichlet 분포

`P ~ Dirichlet(α)`, `α = base_support × conc`:

```
E[P_k] = α_k / Σα = base_k          (기대값은 항상 기준 지지율)
Var(P_k) = base_k(1−base_k) / (conc + 1)
```

- **합=1 보장**: Dirichlet는 단체(simplex) 위에 있어 지지율 합이 항상 1 → 정합성 자동 보장
- **conc 선택**: 후보 지지율의 투표소 간 표준편차 σ_p 로 역산
  `conc ≈ base_k(1−base_k)/σ_p² − 1`
  예) base_A=0.75, σ_p ≈ 2%p → conc ≈ 0.1875/0.0004 − 1 ≈ **468** (코드 기본 500과 근사)
- conc가 작으면 지지율이 극단(0 또는 1)으로 튀어 비현실적. conc ≥ 200 권장.

### 3. 지배 변수는 분포 모수가 아니라 **투표소 수 M (생일 효과)**

검증 실험 결과(20,000회):

| 투표소 수 M | P(쌍둥이 투표소) |
|---|---|
| 50 | 2.2% |
| 100 | 8.7% |
| 200 | 27.5% |
| 400 | 72.9% |
| **860 (전남)** | **99.8%** |

지지율 격차는 거의 영향 없음 (모두 ~99%):

| 지지율 (A vs B) | P(쌍둥이 투표소) |
|---|---|
| 75 vs 15 (압승) | 99.80% |
| 45 vs 45 (접전) | 98.82% |
| 40 vs 35 | 99.75% |

해석: 충돌 확률은 **생일 문제**를 따릅니다.
가능한 `(A,B)` 쌍의 유효 가짓수를 R 이라 하면, M개 투표소 중 한 쌍이라도 충돌할 확률은
대략 `1 − exp(−M² / 2R)`. 전남은 M≈860이라 R이 수십만이어도 충돌이 거의 확실합니다.
흥미롭게도 **압승(75 vs 15)이 접전보다 충돌이 약간 더 잦습니다** — B 득표(15%)가 작은 값 구간에
몰려 (A,B) 쌍의 유효 분산 R이 작아지기 때문. 즉 **분산이 작을수록(κ·conc 클수록) 충돌↑**.
따라서 결과를 좌우하는 건 `base_support` 격차가 아니라 **투표소 수와 투표소 규모 분포**이며,
이것들은 모두 실제 데이터(상수)이므로 시뮬레이션 답은 매우 견고(robust)합니다.
