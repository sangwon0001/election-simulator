# 쌍둥이 투표동 시뮬레이터

제9회 전국동시지방선거(2026-06-03) 시·도지사선거에서, 서로 다른 **두 개표단위(읍면동×구분)**가
A·B 두 후보 득표수가 **동시에 똑같은** '쌍둥이 투표동'이 발생할 확률을 추정한다.
실제 중앙선관위 데이터를 크롤링해 경험적 관측과 몬테카를로 시뮬레이션을 비교한다.

> 쌍둥이 = 투표소(투표구)가 아니라 **개표단위(동×구분)** 기준. 두 단위 i, j에서
> `A_i == A_j` **그리고** `B_i == B_j` 인 충돌(생일 문제 구조).

## 방법론

- **단위**: 읍면동 × 구분(관내사전투표 / 선거일투표). 거소·관외사전·합계는 제외.
- **A, B 후보**: 시·도별 총득표 상위 2명 자동 식별 (전남은 민형배/이정현).
- **기본 모델(권장)**: 동별 **실제 투표수 V·실제 지지율 p**로 `A~Binomial(V, pA)` 추출 →
  모든 쌍 중 (A,B) 동시 일치 카운트. 커스텀 파라미터 0개, 교과서 귀무가설.
- **충돌 탐지**: `(A,B)`를 정수키 `A·BIG+B`로 인코딩 → 정렬 → 중복 카운트. O(M log M).

## 핵심 결과

### 전남도지사 (A=민형배 80%, B=이정현 12%)

- **실제 관측: 동 단위 0쌍, 동×구분 4쌍** (모두 관내사전투표):
  여수 삼일동=신안 하의면, 장성 북하면=함평 엄다면, 보성 노동면=신안 팔금면, 화순 이양면=강진 병영면
- **기본 모델 P(4쌍 이상) ≈ 2.5%**, 기대 ~1.1쌍 → 4쌍은 상위 꼬리의 "운 따른 해".

### 17개 시·도 비교 — "압승만으론 안 되고, 농촌 소규모가 필수"

| 시·도 | A점유율 | 실제 쌍 | 시뮬 기대 |
|---|---|---|---|
| **전남** | 77% | **4** | 1.20 |
| 경북 | 67% | 1 | 0.50 |
| 전북 | 50% | 1 | 0.45 |
| 경남 | 51% | 1 | 0.28 |
| **광주** | **76%** | **0** | 0.02 |
| 경기·서울·기타 | ~50% | 0 | <0.13 |

- **광주는 전남과 같은 압승(76%)인데 쌍둥이 0쌍** — 도시라 동이 크고 득표수가 넓게 퍼짐.
- 전남이 유일한 outlier인 이유 = **3중 조합**:
  ① 압승(B 득표가 수십 표에 뭉침) ② 농촌 소규모 면(작은 단위 多)
  ③ 극단적 사전투표 쏠림(관내사전 51%, 전국 유일 >50% → 선거일 단위까지 잘게 쪼개짐)

## 엔진 메모 (`twin_vote_sim.py`)

- 단위별 `turnout_mean_per_unit`, `support_per_unit`, `alpha_per_unit` 배열 지원.
- `dispersion`(φ): sub-binomial(φ<1) 정규근사. 동전던지기보다 변동 작은 실제 투표 모사용.
- `SimResult.p_exactly(k)`, `p_at_least(k)`, `pair_distribution()`: 쌍 개수 확률.
- 다항분포는 이항분해(`X_A~Bin(V,pA)`, `X_B~Bin(V-X_A, pB/(1-pA))`)로 완전 벡터화.

## 진단으로 밝힌 모델의 한계

근접쌍 분포(`diagnose.py`)를 보면 bulk(거리 d≥2)는 실제와 일치하나, **정확일치(d≤1)에서 실제가
모델의 ~3배**. 분산을 키우거나(conc↓) 줄여도(φ) 이 정확-피크는 재현 불가 — 모델이 못 만드는
**이산적 군집**(인구학적으로 닮은 사전투표 면들)이 실재함을 시사. 다만 그 효과 크기(~2쌍)는
포아송 노이즈와 구분하기 어려워, 결국 **전남 4쌍은 구조가 아니라 상위 운**으로 결론.
